# Parked Epics

## CMS and Authoring

- Explore file-first authoring improvements before committing to a browser CMS.
- If a CMS is added, keep it Git-backed and avoid adding a separate content database.

## Preview and Staging Infrastructure

- Decide how `develop`, Netlify previews, and feature branches should interact.
- Keep preview infrastructure useful for review and validation without creating release confusion.

## Information Architecture

- Revisit the long-term role of `/timeline`, project detail pages, and durable permalinks.
- Keep the site easy to navigate without adding a lot of sections too early.
